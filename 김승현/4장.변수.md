# <span style='font-weight:bold;'> 변수</span>

## <span style='font-weight:bold;'> 1. 변수란 무엇인가? 왜 필요한가?</span>

<span style='color:grey'>모든 애플리케이션은 데이터를 입력받아 처리하고 결과를 출력하는 과정을 거치는데, 이러한 데이터를 관리하기 위한 핵심 개념이 변수입니다.

메모리의 임의의 위치(메모리 주소)에 데이터가 저장되고 CPU는 이 값을 읽어들여 연산을 수행합니다. 하지만 메모리 주소는 코드가 실행될 때 메모리의 상황에 따라 임의로 결정되며 동일한 컴퓨터에서 동일한 코드를 실행하더라도 값이 저장될 메모리 주소는 변경됩니다. 따라서 메모리 주소를 통해 값에 직접 접근하는 것은 올바른 방법이 아닙니다.

이러한 때에 <span style='color:skyblue;'>변수</span>를 사용하는 것입니다.
프로그래밍 언어는 기억하고 싶은 값을 메모리에 저장하고, 저장된 값을 읽어서 재사용하기 위해서 변수라는 메커니즘을 제공하고 있습니다.</span>

</br></br>

## <span style='font-weight:bold;'> 2. 변수 선언</span>

변수는 반드시 선언을 해준 후 사용할 수 있습니다.
현재 변수 선언은 var, let, const로 3가지 <span style='color:skyblue;'>키워드</span>가 있습니다.

es6 도입 전까지는 var키워드만이 변수를 선언하는 유일한 키워드였으나, var의 여러가지 단점(대표적 : 함수 레벨 스코프만 지원)을 보충하기 위해 let, const가 생겨나게 되었습니다.

</br></br>

## <span style='font-weight:bold;'>3. 변수 선언의 실행 시점과 변수 호이스팅</span>

자바스크립트 코드는 인터프리터에 의해 환 줄씩 순차적으로 실행됩니다. 하지만 자바스크립트 엔진은 소스코드 실행을 위한 준비단계인 소스코드의 평가 과정에서 자바스크립트 엔진은 변수 선언을 포함한 선언문(변수 선언문, 함수 선언문 등)을 소스코드에서 찾아내어 먼저 실행합니다. 이 과정이 끝나면 모든 선언문을 제외하고 소스코드를 한 줄씩 순차적으로 실행합니다.

자바스크립트 엔진은 변수 선언이 소스코드의 어디에 있든 상관없이 다른 코드보다 먼저 실행합니다.

    런타임(runtime)
    : 소스코드가 한 줄씩 순차적으로 실행되는 시점

</br></br>

## <span style='font-weight:bold;'> 4. 식별자 </span>

변수 이름을 식별자라고도 합니다.
식별자는 어떤 값을 구분해서 식별할 수 있는 고유한 이름을 말합니다.
값은 메모리 공간에 저장되어있기 때문에, 식별자는 값이 저장되어 있는 메모리 주소와 매핑 관계를 맺으며, 이 매핑 정보도 메모리에 저장되어야 합니다.

변수 외에도 함수, 클래스 등의 이름은 모두 식별자라고 할 수 있으며, 메모리 상에 존재하는 어떤 값을 식별할 수 있는 이름은 모두 식별자라고 부릅니다.

이러한 식별자는 네이밍 규칙을 준수해야하며, 선언을 통해 자바스크립트 엔진에 식별자의 존재를 알리게 됩니다.

</br></br>

## <span style='font-weight:bold;'>5. 식별자 네이밍 규칙</span>

- 특수문자를 제외한 문자, 숫자, 언더바(\_), 달러($)를 포함할 수 있다.
- 숫자로 시작하는 것은 허용되지 않는다.
- <span style='color:skyblue;'>예약어</span>는 식별자로 사용할 수 없다.

  변수도 식별자이기 때문에 위 네이밍 규칙을 따라야합니다.

        * 다음 식별자는 규칙에 위배되므로, 사용할 수 없습니다.

        var first-name;
        var 1st;
        var this;

  ***

        * 대소문자는 구별하므로, 다음 변수는 각각 별개의 변수입니다.

        var firstname;
        var firstName;
        var FIRSTNAME;

  ***

        * 명확하게 이해할 수 있는 이름으로 지어야합니다.

        var x = 100; (잘못된 예)
        var score = 100; (올바른 예)

  ***

        * 변수 선언에 별도의 주석이 필요하다면 변수의 존재 목적을 명확히 드러내지 못하는 것일 수 있습니다.


        (잘못된 예)
        // 경과 시간. 단위는 날짜다.
        var d;

        (옳은 예)
        var elapsedTimeInDays;

  ***

        * 네이밍 컨벤션 : 하나이상의 단어로 구성된 식별자를 만들 때 사용되는 명명 규칙

        1. 카멜 케이스
        var firstName;

        2. 스네이크 케이스
        var first_name;

        3. 파스칼 케이스
        var FirstName;

        4. 헝가리언 케이스
        var strFirstName;
        var $elem = document.getElementById('myId');
        var observalbe$ = fromEvent(document, 'click');

        일관성을 유지한다면 어떤 컨벤션을 사용해도 문제는 없으나, 자바스크립트에서는 일반적으로 변수, 함수에는 카멜케이스를 / 생성자 함수, 클래스 이름에는 파스칼 케이스를 사용합니다. 따라서 코드 전체의 가독성을 높이려면 카멜 케이스와 파스칼 케이스를 따르는 것이 유리합니다.

</br></br></br>

# <span style='font-weight:bold;'>용어 정리</span>

    변수(variable)
    : 하나의 값을 저장하기 위해 확보한 메모리 공간 자체 또는 그 메모리 공간을 식별하기 위해 붙인 이름

    <관련 용어>
    할당:변수에 값을 저장하는 것
    참조:변수에 저장된 값을 읽어 들이는 것

---

    식별자
    : 어떤 값을 구분해서 식별할 수 있는 고유한 이름을 말합니다. 값은 메모리 공간에 저장되어있기 때문에, 식별자는 값이 저장되어있는 메모리 주소와 매핑 관계를 맺으며, 이 매핑 정보도 메모리에 저장되어야합니다. 변수, 함수, 클래스 등 메모리 상에 존재하는 어떤 값을 식별할 수 있는 이름은 모두 식별자라고 부릅니다. 선언을 통해 자바스크립트 엔진에 식별자의 존재를 알립니다. 식별자는 네이밍 규칙을 준수해야하며 값의 의미를 파악할 수 있는 가독성 높은 이름으로 지어야합니다.

---

    키워드(keyword)
    : 자바스크립트 코드를 해석하고 실행하는 자바스크립트 엔진이 수행할 동작을 규정한 일종의 명령어로, 키워드를 만나면 자신이 수행해야 할 약속된 동작을 수행합니다. 예를 들어, var 키워드를 만나면 자바스크립트 엔진은 뒤에 오는 변수 이름으로 새로운 변수를 선언합니다.

    예)  var score;

---

    런타임(runtime): 소스코드가 한 줄씩 순차적으로 실행되는 시점

---

    예약어 : 프로그래밍 언어에서 사용되고 있거나 사용될 예정인 단어

    예) await, break, case, catch, class, const, continue, enum, function ...
